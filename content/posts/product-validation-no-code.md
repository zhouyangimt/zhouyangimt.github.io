## 你花三个月写的产品，发出来没人用

这种事我经历过不止一次，身边做独立开发的朋友也几乎都踩过这个坑。

- 花两周调研技术栈，选了最新的框架
- 花一个月写代码，前后端全自己撸
- 花一周部署上线，配置域名、SSL、支付
- 发到 Product Hunt → 3 个 upvote，其中 2 个是朋友点的
- 发到 Twitter → 5 个 like，0 个注册

**问题出在哪？** 出在你写第一行代码之前。

你从来没有确认过：真的有人需要这个东西吗？他们愿意付钱吗？

这就是产品验证要解决的问题。而且这个过程不需要写任何代码。

---

## 产品验证的本质

一句话：**在投入大量时间写代码之前，用最小的成本确认需求真实存在。**

![产品验证流程图](/images/product-validation-flow.png)

上图是整个验证流程的概览。每一步都是低成本的，目的是快速筛掉坏点子，只把时间花在真正值得做的事情上。

关键心理建设：

> 验证的目标不是「证明我的想法是对的」，而是「尽快发现我的想法是错的」。

抱着这个心态，每一次「被否定」都是赚到了——你省下了三个月的时间。

---

## 第一步：问题访谈

找一个有明确痛点的人，或者你想做的产品可能的用户，直接聊天。

**不要问「你会用这样的产品吗」**——这种问题100%得到「会啊」，毫无意义。

要问的是：

- 「你现在怎么解决这个问题的？」
- 「这个问题多久出现一次？」
- 「你为这个事花过钱吗？花了多少？」
- 「现有的方案哪里让你不爽？」

举一个真实的验证对话。假设你想做一个「AI生成周报」的工具：

```text
你：你平时周报怎么写？
对方：周五下班前花20分钟憋。
你：那挺痛苦的，有没有用过什么工具？
对方：用过几个模板，都不太对味，还是自己写。
你：试过用AI写吗？
对方：试过ChatGPT，但每次要描述很多上下文，也麻烦。
你：如果有人做了一键生成周报的工具，你愿意试试吗？
对方：可以试试，但得看生成的像不像我写的。
你：如果收费的话，你觉得多少钱合理？
对方：emmm...如果不贵的话，一个月十几块吧。
```

这段对话已经提供了大量信息：
1. 痛点真实存在（每周五花20分钟）
2. 现有方案不满意（ChatGPT太麻烦，模板不对味）
3. 有付费意愿（但不高，月费十几块）
4. 隐含的深层需求：生成内容要像自己写的（个性化）

**访谈目标：5-10个人。** 如果其中3个以上给了你正面的信号（痛点明确 + 表达付费意愿），进入下一步。

---

## 第二步：烟雾测试（Smoke Test）

烟雾测试的本质是：**假装产品已经存在，看有没有人点击。**

### 做法

1. 用 Carrd / Notion / Google Sites 搭一个单页 Landing Page（30分钟）
2. 写清楚产品是什么、解决什么问题
3. 放一个大大的 CTA 按钮：「抢先体验」「加入等待列表」
4. 按钮背后连接一个 Google Form / Tally 表单，收集邮箱
5. 把页面链接发到目标用户聚集的地方

### Landing Page 写什么？

不要写功能列表，写「你能得到什么」。

对比一下：

| 错误（写功能） | 正确（写价值） |
|---|---|
| 支持 GPT-4 和 Claude 两个模型 | 生成的内容读起来像你亲手写的 |
| 拖拽式模板编辑器 | 30秒搭好一个页面，不用学设计 |
| 自动同步日历 | 永远不会忘记重要的事 |

### 去哪里推广？

取决于你的目标用户在哪：

| 目标用户 | 推广渠道 |
|---|---|
| 独立开发者 | Twitter/X、V2EX、Indie Hackers |
| 职场白领 | 即刻、脉脉、小红书 |
| 海外用户 | Reddit、Hacker News、Product Hunt |
| 设计师 | Dribbble、小红书设计分区 |

**判断标准**：

- 邮箱注册率 < 3%：这个点子可能不行，或者 Landing Page 表达有问题
- 邮箱注册率 5%-10%：有戏，继续验证
- 邮箱注册率 > 10%：赶紧做

---

## 第三步：假门测试（Fake Door Test）

烟雾测试验证了「有人感兴趣」。但**感兴趣 ≠ 愿意付钱**。

假门测试更狠：直接放一个付费按钮，看有多少人点。

### 具体做法

在 Landing Page 上增加一个「立即购买」或「开始免费试用（需绑卡）」的按钮。

用户点击后，不要真的收款，而是跳转到一个页面：

```html
<!-- fake-door.html -->
<h2>感谢你的兴趣！</h2>
<p>我们正在做最后的上线准备，你是第 37 位等待的用户。</p>
<p>请留下邮箱，上线后第一时间通知你。</p>
<!-- 一个邮箱输入框 -->
```

或者更诚实一点：

```html
<h2>这个功能还在开发中</h2>
<p>看起来你对这个产品感兴趣。我们正在根据反馈调整，能不能花2分钟聊聊你的需求？</p>
<a href="你的日历链接">预约10分钟通话</a>
```

### 为什么要这么做？

- 点击付费按钮的人，是真的有付费意愿的
- 跳转页面帮你收集了最高质量的用户线索
- 你还可以直接约他们聊，获取最真实的反馈

**判断标准**：

- 付费点击率 > 2%（相对于页面总访客）：需求强度和付费意愿都验证通过
- 付费点击率 < 1%：用户只好奇，不愿掏钱 → 要么定价问题，要么需求不够痛

## 第四步：人工MVP（Concierge MVP）

这是我最推荐的一步。也是省掉最多无效代码的一步。

**人工MVP** 的意思是：你手动完成产品的核心功能，不写代码。

### 例子

| 产品想法 | 人工MVP做法 |
|---|---|
| 自动生成周报的AI工具 | 你手动帮用户写周报，收他们钱 |
| 帮你找最优机票的App | 你手动比价，发结果给他们 |
| 自动化的数据分析看板 | 你用Excel做好，截图发给他们 |
| 帮你写邮件的AI助手 | 你在ChatGPT里写好，发给他们 |

### 为什么这么做？

因为你要验证的是 **「结果的交付」** 到底有没有价值，而不是你的产品 UI 好不好看。

如果手动交付的结果用户满意、愿意继续付费，那说明核心价值成立。这时候写代码做自动化才有意义。

如果手动交付的结果用户不满意——问题不在 UI，不在速度，而在于**这件事本身就不值得花钱**。

### 真实案例：我见过的一个开发者

他想做一个「自动生成API文档的工具」。做了两个月，功能很齐全。

后来他复盘时说：如果当初先做一个「人工MVP」——用户把代码给他，他手动写成文档收$50——他会立刻发现：**大部分团队根本不需要第三方工具生成API文档，用Swagger就够了。**

两个月白干。$0收入。

## 第五步：验证通过后的判断框架

当你走完以上四步，你手里应该有：

1. 至少5个人的访谈记录，确认痛点真实
2. Landing Page 转化数据（邮箱收集数 / 页面访客数）
3. 假门测试的付费意愿数据
4. 人工MVP的用户满意度反馈

现在做一个决策：

```
把所有信号汇总打分（1-5分）：
- 痛点明确度：___
- 付费意愿：___
- 你能触达用户：___
- 你有能力解决：___
- 市场够小（大公司看不上）：___

总分 ≥ 20：全力做
总分 15-19：先做一个更小的MVP再测
总分 < 15：换方向
```

**不需要每一项都是满分。** 但总分不够就别骗自己。

## 工具清单：零成本验证工具箱

整个验证流程不需要花一分钱：

| 用途 | 工具 | 费用 |
|---|---|---|
| Landing Page | Carrd.co | 免费 |
| 邮箱收集 | Google Form / Tally | 免费 |
| 邮件发送 | Mailchimp 免费版 | 免费1000人 |
| 预约通话 | Calendly 免费版 | 免费 |
| 用户访谈 | 微信语音 / Google Meet | 免费 |
| 跟踪分析 | Google Analytics | 免费 |
| 表单 | Google Forms | 免费 |

不需要服务器、不需要域名、不需要写代码。

## 一篇文章总结

| 验证步骤 | 做什么 | 成本 | 判断标准 |
|---|---|---|---|
| 问题访谈 | 聊5-10个目标用户 | 几个小时 | 3人以上有明确痛点 |
| 烟雾测试 | Landing Page + 邮箱收集 | 30分钟搭页面 + 推广 | 注册率 > 5% |
| 假门测试 | 放付费按钮看点击 | 改一行HTML | 付费点击率 > 2% |
| 人工MVP | 手动交付核心价值 | 每天1-2小时 | 用户满意 + 愿意继续付费 |

**核心原则：验证一次只需要花你几小时到一两天。** 如果方向不对，你损失的不是三个月，而是一天。
